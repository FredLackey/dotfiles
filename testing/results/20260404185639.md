## Errors

### Error 1: wget Dependency Not Met for acli.sh

- **Phase**: acli.sh installer (DEV category, step 3 of 9 in Core CLI Tools)
- **Exit Code**: 1
- **Error Message**:
```
Error: wget is required to install Atlassian CLI.
```
- **Root Cause**: `setup.sh` calls `acli.sh` on line 65, but `wget.sh` is not called until line 68. The `acli.sh` script correctly checks for `wget` as a dependency, but the orchestrator (`setup.sh`) places `acli.sh` before `wget.sh` in the execution order. This is an installer sequencing bug in `setup.sh`, not a bug in `acli.sh` itself.
- **Relevant code in src/os/ubuntu-server/setup.sh** (lines 62-68):
  ```bash
  run_installer "git.sh" "DEV"
  run_installer "github-cli.sh" "DEV"
  run_installer "acli.sh" "DEV"       # <-- called here, wget not available yet
  run_installer "curl.sh" "UTILS"
  run_installer "wget.sh" "UTILS"     # <-- wget installed here, too late
  run_installer "gpg.sh" "UTILS"
  ```
- **Suggested Fix**: Move `wget.sh` (and `curl.sh`) before `acli.sh` in `setup.sh`. Both curl and wget are foundational download utilities that should be installed as early UTILS prerequisites, before any installer that depends on them. The simplest fix is to move `run_installer "wget.sh" "UTILS"` and `run_installer "curl.sh" "UTILS"` to run immediately after `build-essential.sh`, before any DEV-category installers.

## Recommendations

1. **Fix the installer ordering in setup.sh (critical)**: Move `wget.sh` and `curl.sh` to run before `acli.sh`. These are fundamental download utilities. A clean ordering would be:
   - After `build-essential.sh`, install `curl.sh`, `wget.sh`, and `gpg.sh` as a "download tools" block.
   - Then proceed with DEV tools (`git.sh`, `github-cli.sh`, `acli.sh`, etc.) that may depend on those utilities.

2. **Consider adding wget to the Docker base image**: The Dockerfile already includes `curl` as a base package. Adding `wget` would mirror real-world Ubuntu server defaults and prevent this class of failure in future installers.

3. **Audit all other installers for implicit dependency ordering assumptions**: Now that `acli.sh` revealed this pattern, it's worth reviewing all installers that declare dependencies on curl, wget, gpg, or other utilities to confirm they are all ordered after those utilities are installed.

4. **Consider a dependency graph or explicit prerequisite block**: The current sequential list in `setup.sh` makes it easy to accidentally place tools out of order. Adding a clearly labeled "# Prerequisites - must run first" block at the top of `install_applications()` could prevent this class of bug.

## Troubleshooting Results

### Issues Identified

**Issue 1: Installer sequencing bug — acli.sh placed before wget.sh and gpg.sh**

- Root Cause: In `src/os/ubuntu-server/setup.sh`, the "Core CLI Tools" block called `acli.sh` at line 65, before `curl.sh` (line 66), `wget.sh` (line 67), and `gpg.sh` (line 68). The `acli.sh` installer correctly checks for `wget` and `gpg` as prerequisites and exits with code 1 when they are absent. Because `setup.sh` runs with `set -e`, this hard-stopped the entire setup before any further installers could run.
- File Modified: `src/os/ubuntu-server/setup.sh`
- Fix Applied: Reordered the "Core CLI Tools" block so that `curl.sh`, `wget.sh`, and `gpg.sh` run first, followed by `git.sh`, `github-cli.sh`, and `acli.sh`. Added an inline comment explaining why the download utilities must come first.
