### Warning 1: NVM PATH not reloaded in session
- **Phase**: nvm.sh
- **Message**: `Warning: Failed to load NVM into memory. npm-dependent installs may fail.`
- **Analysis**: NVM installs to `~/.nvm` and modifies `.bashrc`, but in-session PATH sourcing in a non-interactive shell requires explicit sourcing. The `setup.sh` does source `nvm.sh` after installation, so subsequent installers work correctly. This warning is cosmetic.

### Warning 2: AWS CLI QEMU warning on ARM64
- **Phase**: aws-cli.sh
- **Message**: `qemu-x86_64: Could not open '/lib64/ld-linux-x86-64.so.2': No such file or directory`
- **Analysis**: The Docker host is an Apple Silicon Mac (ARM64). The AWS CLI installer downloads an x86_64 binary. QEMU tries to translate it but lacks the x86_64 linker. Despite this warning, the installer reports success and places the binary at `/usr/local/bin/aws`. The binary may not be functional on ARM64. This is a host architecture issue, not an installer defect.

### Warning 3: Gitego upstream go.mod incompatibility

- **Analysis**: The `bgreenwell/gitego` upstream project declares a Go version in its `go.mod` that exceeds the version available from Ubuntu 22.04 apt (`golang-go` provides 1.18.x). The installer exits 0 gracefully. No action needed; this is an upstream issue.

## Recommendations

### Upstream Go Compatibility (td, sidecar, nightshift, gitego)

All four `go install` tools (td, sidecar, nightshift, gitego) fail because Ubuntu 22.04's apt package for `golang-go` provides Go 1.18.x, while all four upstream `go.mod` files declare minimum versions of 1.24.0 or higher. The graceful-skip pattern is the correct short-term fix.

Long-term options:
1. Install Go from the official Go tarball (golang.org) instead of apt to get a current version (1.21+). This would enable successful installation of all four tools.
2. Upgrade the base Ubuntu image to 24.04, which ships with a newer Go version in apt.
3. Accept the skips as permanent for Ubuntu Server and document the tools as "not available without manual Go upgrade."

### AWS CLI on ARM64
The AWS CLI install emits a QEMU warning because the container runs on an Apple Silicon host (ARM64) and the install script fetches an x86_64 binary. On real AMD64 Ubuntu servers this warning does not appear and the binary functions correctly. No code change is needed.

### Git Identity Not Configured
Expected in containers. Not actionable in setup scripts — git identity must be set by the user after installation.
